# Changelog - 260519

## 260519 - Per-Task Model Selection

### Summary
Added optional per-task model selection for the underlying agent CLIs. Reviews can now run on a smaller / faster model (e.g. `claude-sonnet-4-6`) while dev / create stays on a stronger model (e.g. `claude-opus-4-7[1m]`). Codex flows pick up the same flag.

### Added
- `model` field on per-task and complexity-override entries in `agentConfig`.
- Top-level `defaultModel` applied to every task unless overridden.
- `tmux-wrapper build-cmd` accepts `--model <id>` and injects `--model` into both `claude --dangerously-skip-permissions` and `codex exec` commands.
- `tmux-wrapper agent-cli` accepts `--model <id>`.
- `orchestrator-helper agents-resolve` and `retro-agent` return the resolved `model` so step scripts can pass it through.
- Documentation refresh in `data/agent-config-prompts.md` showing the new 3-tuple Custom syntax (`task: primary, fallback[, model]`).
- 22 new regression tests in `tests/test_agent_config_model.py` covering core resolution, agents-file generation, state-doc serialization (including sentinel omission), retro fallback, `build-cmd` flag injection for both Claude and Codex, and fail-fast on `--model` without a value.
- Workflow step files (`data/retry-fallback-strategy.md`, `data/retry-fallback-implementation.md`, `data/code-review-loop.md`, `steps-c/step-03-execute.md`, `steps-c/step-03a-execute-review.md`, `steps-c/step-03b-execute-finish.md`) now thread `.model` from `agents-resolve` / `retro-agent` into `tmux-wrapper build-cmd` so create / dev / auto / review / retro sessions actually use the configured model. `data/scripts-reference.md` documents the new `.model` field.

### Changed
- `resolve_agent_for_task` and `resolve_agent` now return a 3-tuple `(primary, fallback, model)`; updated callers and the one existing assertion in `tests/test_runtime_layout.py`.
- `agent_cli(agent)` now accepts an optional `model` argument and shell-quotes the model ID (handles `[1m]` style suffix).
- State document serialization in `commands/state.py` writes `model` lines under `perTask` and `complexityOverrides`, plus an optional top-level `defaultModel`. Sentinel values are filtered through `_model_or_none` and never persisted.
- `tmux-wrapper agent-cli --model` now fails fast (exit 1, "--model requires a value") instead of silently accepting a bare flag.
- `data/agent-config-prompts.md` enumerates the full sentinel set (`auto`, `default`, `false`, `none`, `null`) and notes they are stripped from persisted state.

### Backward Compatibility
- All existing schemas remain valid: `model` is optional everywhere; when absent the CLI command is identical to today.
- Sentinel values `""`, `"auto"`, `"default"`, `"false"`, `"none"`, `"null"` are treated as unset.

### Second review pass (2026-05-20)

Addresses bma-d's CHANGES_REQUESTED follow-up and CodeRabbit's second round.

- **Agent-aware model resolution.** The retry/fallback helper now exposes `should_apply_primary_model "$current_agent"`; `--model` is included only when the current attempt is on the primary agent. Fallback retries no longer inherit the primary model. Updated in every step that spawns: `data/retry-fallback-implementation.md`, `steps-c/step-03-execute.md` (create + dev), `steps-c/step-03a-execute-review.md` (auto), `data/code-review-loop.md` (review). Retro keeps its own `--model "$retro_model"` branch.
- **Argv-safety.** All `--model` flags now use a quoted `"$primary_model"` (or `"$retro_model"`) form rather than an unquoted prebuilt shell fragment, so bracketed IDs like `claude-opus-4-7[1m]` reach the CLI as one literal argv element even when a same-prefix filename exists in cwd.
- **Smoke contract.** `scripts/smoke-test.sh` updated to match the new `build-cmd` literal strings in `code-review-loop.md`, `step-03-execute.md`, `step-03a-execute-review.md`, and `retry-fallback-implementation.md`. `npm run test:smoke` is green.
- **Coverage gap closed.** New `MarkdownHandoffShellContractTests` in `tests/test_agent_config_model.py` runs the bash pattern from the markdown snippets in a subprocess and asserts: (a) primary attempt forwards the configured model, (b) fallback attempt does **not**, (c) `claude-opus-4-7[1m]` reaches Python as one literal argv element even with a matching `claude-opus-4-71` file in cwd, (d) absent model means no `--model` flag. Snippet is POSIX-compatible (works on bash 3.2 too).
- **Ruff RUF059.** Renamed unused `fallback` unpacks to `_fallback` in `tests/test_agent_config_model.py` to silence the lint.
